# POS Modernization Roadmap

## Goal
Turn POS Awesome from a strong ERP-linked POS into a more modern retail/hospitality product with clearer differentiation against Odoo-style POS flows and other open-source POS systems.

## Phases
- [x] Phase 1: Inspect current codebase capabilities and UX structure
- [x] Phase 2: Research current POS market and open-source feature baselines
- [x] Phase 3: Identify product gaps and translate them into implementation themes
- [x] Phase 4: Produce a prioritized roadmap with effort and impact
- [x] Phase 5: Convert roadmap into a repo-specific phased implementation plan

## Decisions
- Prioritize product gaps that materially change buying decisions: self-order, kitchen/prep workflow, restaurant ops, stored value, and cashier ergonomics.
- Avoid recommending features already strong in the app such as offline sync, split payments, loyalty/coupons, multi-currency, weighted barcodes, and cash movement.
- Save the implementation plan as a single master roadmap document, but execute it as separate PR phases.
